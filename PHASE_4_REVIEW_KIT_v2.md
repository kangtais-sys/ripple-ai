# 2. Use Cases (per permission — 각 권한 신청 칸에 해당 박스만 붙여넣기)

## 2.1 `instagram_business_basic`

```
Ssobi requires this permission to identify the Instagram Business
account a creator has connected and to display it as the active asset
selector throughout the app.

Use case flow:
  1. User signs into Ssobi.
  2. User taps "Connect Instagram" → Instagram OAuth consent screen.
  3. After consent, our backend calls GET /me with this permission to
     retrieve id, username, and account_type.
  4. The account is persisted in our user-isolated database (ig_accounts,
     secured by Supabase Row-Level Security).
  5. @millimilli.kr is rendered in the app header on every screen and
     is used as the asset for all subsequent operations.

Without this permission no downstream feature is possible.

Data handling:
  • Stored only in the user's own row, accessible only via RLS.
  • User can disconnect at Profile → Account → Instagram → Disconnect,
    which revokes the token and deletes the record.
  • Full account deletion: /privacy#data-deletion (data removed in 7 days).

Screencast: Video 1 (Sign-in + OAuth + connection).
```

## 2.2 `instagram_business_manage_comments`

```
Ssobi uses this permission to read incoming comments on the connected
account's media and post replies on the account owner's behalf — only
after the owner reviews and explicitly approves each draft.

Use case flow:
  1. A new comment arrives on the owner's media.
  2. Our system reads the comment via GET /{comment_id}.
  3. The comment is classified (urgent, business, normal) and an AI
     reply draft is generated by Anthropic Claude using the owner's
     learned tone profile.
  4. The draft appears in the owner's "Auto Reply → Inbox" dashboard.
  5. The owner reviews the draft, can edit it, and clicks "Approve · Send"
     or "Manual reply" (takeover).
  6. On approval, our backend calls POST /{comment_id}/replies via the
     Instagram Graph API.
  7. Every event is logged in our reply_logs audit table.

K-Beauty creators receive hundreds of repetitive comments daily
("Where to buy?", "What's the price?"). Ssobi automates the friendly
drafting so creators can focus on creative work while keeping audience
engagement.

User controls:
  • Master auto-reply toggle (owner can disable any time)
  • Per-channel toggle (comments independently from DMs)
  • Per-thread "Manual reply" button — owner hands off to human agent
  • Audit log of every event (drafted, approved, sent, skipped)
  • Banned-word list — flagged drafts route to manual review

Screencast: Video 2 (Sign-in + OAuth + inbox + AI draft + approve + send action).
```

## 2.3 `instagram_business_manage_messages`

```
Ssobi uses this permission to read direct messages sent to the connected
account and reply on the owner's behalf, strictly within Meta's
24-hour Standard Messaging Window, only after the owner approves each
draft.

Use case flow:
  1. A user sends a DM to the owner's Instagram Business account.
  2. Our system reads the message via the conversations endpoint.
  3. An AI reply draft is generated by Anthropic Claude in the owner's
     learned tone.
  4. The draft appears in the DM inbox dashboard.
  5. The owner reviews/edits and clicks "Approve · Send" or "Manual
     reply" (takeover to human agent).
  6. On approval, our backend calls POST /me/messages — strictly within
     the 24-hour window unless an approved message tag applies.
  7. The conversation is mirrored in the app's inbox so the owner can
     intervene at any time.

DM inquiries are time-sensitive — creators cannot respond 24/7
manually. Ssobi enables prompt customer service in the creator's voice.

Compliance and user controls:
  • The 24-hour Standard Messaging Window is strictly respected — our
    code blocks any send attempt outside the window.
  • Per-channel toggle (DMs independent of comments)
  • Per-conversation "Manual reply" — owner marks a thread as manual-only
  • Audit log of every event
  • No outbound messaging to non-customers — replies only

Screencast: Video 3 (Sign-in + OAuth + DM inbox + AI draft + approve +
window check + manual-reply demonstration).
```
