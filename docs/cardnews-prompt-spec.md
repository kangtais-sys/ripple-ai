# 카드뉴스 AI 프롬프트 업그레이드 명세서

작성일: 2026-04-25
대상 파일: `src/lib/cardnews-prompt.ts`, `src/app/api/cardnews/generate/route.ts`, `src/app/api/cardnews/fact-check/route.ts`

## 1. 문제 정의

현재 카드뉴스 생성 결과에 두 가지 큰 문제가 있다.

### 1.1 본문에 메타 라벨이 박힘
- "(참고용)" / "(검증 필요)" 같은 시스템 라벨이 본문 텍스트에 그대로 들어감
- 카드뉴스 가독성·완성도를 떨어뜨림 (촌스러움)
- → 본문에는 절대 박지 않는다. UI 레이어의 issue 패널에만 노출.

**현 코드의 위치**: `fact-check/route.ts` 의 `cleaned` 매핑에서 `text.replace(claim, claim + tag)` (이번 커밋에서 제거 완료)

### 1.2 추천 콘텐츠가 너무 뻔함
실제 사례 — 사용자가 입력: **"커피 싫어하는 사람도 좋아하는 카페 메뉴"**

AI가 생성한 본문:
- "초콜릿 음료"
- "과일 음료"
- "차 종류"

문제:
- 누구나 알고 있는 일반 카테고리
- 저장·공유할 동기 없음 ("이걸 모르는 사람이 어디 있어?")
- 인스타그램 캡션 톤이 아님 (블로그 1990s 톤)

기대했던 결과 (예시):
- "스벅 신메뉴 — 라이트노트 블론드" (특정 시그니처)
- "커피빈 일리 콜드브루 (실제로 우유맛)"
- "블루보틀 인절미 라떼 (한정판)"
- "런던 베이글 뮤지엄 — 얼그레이 라떼"
- "대전 성심당 망고우유 (메뉴 아닌데 카페에서 줌)"
- "이디야 흑임자 모카치노" 같은 구체 음료명 + 어디서 살 수 있는지 + 왜 좋은지

## 2. 핵심 개선 원칙

### 원칙 A — 일반명사 금지, 구체명사 강제
> ❌ "초콜릿 음료" → ✅ "스타벅스 더블 초콜릿 자바칩 프라푸치노"
> ❌ "차 종류" → ✅ "공차 우롱밀크티 80% 단맛"
> ❌ "과일 음료" → ✅ "이디야 토마토 주스 (호불호 갈리지만 진심 추천)"

### 원칙 B — 트렌드 단서 우선
- 최근 30~60일 안에 SNS·블로그·뉴스에 등장한 메뉴/브랜드 우선
- "신메뉴", "한정판", "리뉴얼", "재출시", "인스타에서 화제" 같은 시간성 키워드 활용
- 시즌(봄/여름/가을/겨울) 메뉴는 현재 시즌과 매칭 (system prompt에 `currentSeason` 주입)

### 원칙 C — 시그니처/한정 메뉴 강조
- 일반 메뉴(아메리카노) X / 그 카페에만 있는 시그니처 메뉴 O
- 출시 시기·매장 한정·기간 한정 등 희소성 명시

### 원칙 D — 인스타 톤
- 1990s 블로그 톤 ("커피를 싫어하시는 분도 즐길 수 있는 다양한…") 금지
- MZ 인스타 톤 ("이거 진짜 커피 싫어하는 친구도 인정함")
- 짧고, 단호하고, 과장 없이 솔직

### 원칙 E — 메타 라벨 본문 진입 금지
- "(참고용)", "(검증 필요)", "(주관적)", "[출처:..]" 같은 시스템성 메타텍스트는 본문 진입 절대 금지
- 검증이 필요한 정보는 두 가지 중 택일:
  1. 본문에서 통째로 삭제하고 다른 구체 정보로 교체
  2. UI 레이어의 issue 패널로만 노출 (본문은 깨끗하게)

## 3. 프롬프트 구조 변경안

### 3.1 buildContentGenerationPrompt 시스템 룰 추가

`cardnews-prompt.ts` 의 `buildContentGenerationPrompt` 안에 다음 룰을 추가/강화:

```
[필수 룰 — 위반 시 재생성]

1. 모든 본문 슬라이드는 다음 중 하나의 형식이어야 함:
   (a) 구체 브랜드/메뉴/제품명 + 한 줄 이유
   (b) 구체 장소/이벤트명 + 가격·시간 정보
   (c) 구체 인물·사례 + 결과 수치
   ※ "다양한 종류", "여러 가지", "사람마다 달라요" 등 일반화 금지

2. 본문에 다음 라벨/메타텍스트 절대 사용 금지:
   - "(참고용)", "(검증 필요)", "(개인적 의견)", "(주관적)"
   - "출처:", "[출처]", "*", "주의:"
   - "더 자세한 내용은 댓글로", "DM 주세요" 같은 hollow CTA (CTA 슬라이드에만 허용)

3. 트렌드 우선순위 (1 > 2 > 3):
   1) 최근 30일 내 SNS/뉴스에 등장한 신메뉴·신제품
   2) 시즌 한정·매장 한정 시그니처
   3) 클래식 베스트셀러 (단, 5번 슬라이드 안에 1개 이하)

4. 톤:
   - 친구한테 이야기하듯 (반말/존댓말은 toneStyle 따라)
   - "~라고 합니다", "~라고 생각됩니다" 금지 → 단정적으로 "~임", "~야"
   - 과장된 형용사 금지: "최고의", "절대", "100%", "단연코"

5. 결과의 50% 이상이 구체 고유명사(브랜드·메뉴·장소·인물·책)를 포함해야 함.
   ※ entity 배열에 그 고유명사를 type/name 으로 명시 (이미지 자동 매칭에 사용)
```

### 3.2 트렌드 시드 주입 (researchData 보강)

현재 `researchData` 는 외부에서 주입받는 free-text. 다음을 자동으로 prepend:

```ts
const trendSeed = `
[현재 시점 정보]
- 시즌: ${currentSeason()} (${new Date().getFullYear()}년)
- 카테고리: ${category}
- 자동 트렌드 풀 (있으면 우선 활용):
${getTrendPoolForCategory(category, 5).map(t => `  · ${t}`).join('\n')}

[추천 시드]
${categoryTrendSeeds[category] || categoryTrendSeeds.etc}
`
```

`categoryTrendSeeds` 는 카테고리별로 미리 큐레이션한 30~60일치 트렌드 키워드/사례를 담은 상수. 예:

```ts
const categoryTrendSeeds = {
  cafe: `
시그니처 메뉴 우선:
- 스타벅스 봄 한정 (라이트노트, 벚꽃, 청귤)
- 블루보틀 시즌 (인절미 라떼, 피스타치오)
- 런던 베이글 뮤지엄 (얼그레이/잠봉뵈르)
- 더리터 (우유 대신 두유 옵션 메뉴)
- 매머드 커피 (대용량, 가성비)
- 컴포즈 커피 (프로틴 음료)
지역 카페:
- 성수동·연남동 신생 로스터리
- 부산 영도/광안리 뷰 카페
- 제주 함덕·한림 디저트 카페
2026년 이슈:
- 디카페인 비중 확대
- 호두라떼·흑임자라떼 류 한식 라떼 부상
`,
  // ... 다른 카테고리들
}
```

### 3.3 자체평가 점수 강화

현재 `hook_score`, `viral_score`, `savability_score` 가 있음. 새 점수 추가:

```
specificity_score (0~10):
- 본문 슬라이드 N개 중 구체 고유명사를 포함한 슬라이드 비율
- 5/N 미만이면 5점 미만 → 재생성 트리거
```

`/api/cardnews/generate/route.ts` 에서 `specificity_score < 5` 면 자동 재요청 1회.

### 3.4 entity 필드 강제

본문 슬라이드의 `entities` 배열에 실제 브랜드·메뉴·장소·인물·책이 type/name 으로 들어가야 함.
이건 이미 schema 에 있지만 현재 비어있는 경우 많음 → "각 본문 슬라이드는 entity 1개 이상 필수" 룰 추가.

## 4. fact-check 로직 변경 (이미 적용)

- `risk: medium` 의 action 을 `mark` → `flag` 로 변경
- `cleaned` 에서 본문에 라벨 박지 않음 — 원본 그대로 통과
- UI 레이어 (`createData.factIssues`)에만 노출 → 사용자가 직접 검토하고 수정하도록 유도

## 5. 구현 우선순위

| 우선순위 | 항목 | 예상 작업량 |
|---|---|---|
| P0 | 본문 메타 라벨 진입 금지 룰 (시스템 프롬프트) | 30분 |
| P0 | (참고용)/(검증 필요) 자동 박기 제거 | 완료 ✓ |
| P1 | 일반명사 금지 / 구체명사 강제 룰 | 1시간 |
| P1 | categoryTrendSeeds 상수 작성 (cafe/beauty/food/travel/fashion 5개) | 2~3시간 |
| P2 | specificity_score 자체평가 + 재생성 트리거 | 1시간 |
| P2 | entity 1개 이상 강제 | 30분 |
| P3 | 시즌 자동 주입 (currentSeason 함수) | 30분 |

## 6. 검증 케이스

다음 입력으로 테스트해서 "구체 메뉴명 + 매장명" 조합이 5개 슬라이드 중 4개 이상에 나오는지 확인:

1. "커피 싫어하는 사람도 좋아하는 카페 메뉴" → cafe
2. "민감 피부에 진짜 안 자극적인 토너 5개" → beauty_product
3. "이번 봄 입을 만한 9만원 이하 트렌치코트" → fashion
4. "혼자 가도 어색하지 않은 서울 브런치 카페" → cafe
5. "월급 200만원에서 100만원 모으는 법" → money_tip

각 케이스에서 결과를 검토한 뒤, 일반명사 비율 / 메타 라벨 출현 / 톤 / 저장 동기 4가지 기준으로 채점.
